  <Accordion title="Jumper Earn — Unified Cross-Chain Yield Discovery and Execution">
    ### Overview

    Jumper Earn is a personalized yield discovery and execution engine that enables users to move from idle assets to earning positions through a single, intent-based workflow.

    This experience is powered in part by the vaults.fyi API, which provides standardized yield data, transaction payloads, position tracking, and attribution across DeFi. Combined with Jumper's cross-chain routing, users can discover and deposit into vaults across networks without managing complex multi-step flows.

    ### The Problem: Scattered DeFi Yields

    Despite DeFi's growth, yield opportunities remain fragmented across chains and protocols. Users face several sources of friction:

    - **Discovery fragmentation:** Yield opportunities are spread across dozens of chains and protocols, each with different interfaces and standards.
    - **Asset mismatch:** Users often do not hold the exact asset required by a vault, requiring swaps or bridging before depositing.
    - **Execution complexity:** Moving from an asset on Chain A to a vault on Chain B can require multiple manual steps (approve → swap → bridge → swap → deposit).
    - **Data staleness:** Inaccurate or delayed data can result in failed or suboptimal transactions.

    These frictions reduce conversion from yield discovery to actual deposits and create significant integration overhead for applications attempting to unify the experience.

    ### The Solution: A Unified Intent Engine

    Jumper Earn reduces this complexity by unifying discovery, execution, and tracking into a single flow.

    By integrating the vaults.fyi API, Jumper accesses standardized, real-time yield data across curated opportunities without building and maintaining protocol-specific integrations. This lets Jumper focus on routing, UX, and cross-chain execution while relying on vaults.fyi for normalized yield intelligence.

    The result is an experience that moves users from any asset on any chain to a target vault through a streamlined intent-based workflow.

    ### How Jumper Earn Uses the vaults.fyi API

    **Market & Vault Data**

    Jumper Earn uses vaults.fyi as a data layer for vault metadata and strategy descriptions, real-time and historical APY and TVL, and yield and incentive breakdowns. This standardized data enables consistent comparisons across protocols and removes the need for custom data pipelines per integration.

    **Transactional Support**

    For many supported vaults, vaults.fyi provides transaction payloads for token approvals, deposits, and withdrawals (including multi-step flows where the user enters a redemption queue). These payloads feed into Jumper's routing engine and transaction orchestration logic.

    When execution requires more complex routing — such as cross-chain swaps or multi-step flows — Jumper routes user intents through LI.FI Composer, which bundles actions into a single execution flow.

    **Position Tracking**

    The vaults.fyi position endpoints help power Jumper's portfolio visibility by detecting existing vault positions, tracking deposited balances, and supporting cross-chain portfolio views. This allows Jumper to present a unified yield dashboard without integrating directly with each protocol's indexing logic.

    **Adoption Tracking & Attribution**

    vaults.fyi provides infrastructure for measuring vault distribution and adoption. Using vaults.fyi data and dashboards, Jumper can track inflows originating from Jumper Earn, monitor net TVL associated with the integration, and attribute deposits to specific providers (protocols or curators). This creates a shared attribution layer between integrators driving user flows and providers supplying vault strategies, with a neutral onchain-based source of truth for both sides.

    ### Example User Flow

    1. **Wallet Connection:** Jumper detects assets and existing positions.
    2. **Yield Discovery:** Relevant vaults are surfaced to the user.
    3. **Vault Selection & Execution:** The user selects a vault and confirms the action. Swaps, bridges, and deposits are orchestrated via Jumper routing using vaults.fyi payloads where applicable.
    4. **Post-Execution Tracking:** Positions are tracked using position data from vaults.fyi and other sources.

    ### Results

    - **Reduced integration overhead** — access to multiple protocols through a single API removed the need for many custom integrations.
    - **Improved conversion** — streamlined discovery-to-deposit flows reduced time-to-yield for users.
    - **Enabled measurable distribution** — attribution and TVL tracking improved visibility for both Jumper and vault providers.
    - **Standardized yield data** — consistent data improved comparability and user decision-making.

  <Accordion title="Etherscan Cards — Contextual Yield Insights Inside a Block Explorer">
    ### Overview

    [Etherscan](https://etherscan.io/) is Ethereum's premier block explorer. Etherscan has leveraged the vaults.fyi API to give users more insights into yield opportunities via the Etherscan Cards integration.

    ### The Idea: Contextual Yield Insights for User Addresses

    When users explore an Ethereum address on Etherscan, they're often looking to understand what that wallet holds and how it's being used. The Etherscan Cards integration surfaces what those idle assets *could* be doing.

    By embedding vaults.fyi market data and transactional support, Etherscan delivers real-time DeFi yield opportunities and enables user interaction with vaults directly within the block explorer interface.

    ### How It Works

    1. **Detect Idle Balances:** When a user views an Ethereum wallet address, the Insights API checks for idle balances in ETH and stablecoins (USDC, USDT, and DAI).
    2. **Find Relevant Yield Opportunities:** Etherscan queries the Insights API to identify the top DeFi yield opportunities that match those assets, filtered by clear and trusted criteria.
    3. **Display Vault Details:** The Yield Opportunity card appears in the Cards tab of the wallet page (for example, [vitalik.eth](https://etherscan.io/address/0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045#cards)), highlighting the token type, protocol name, vault name, and estimated 7-day APY.
    4. **Connect the User to the Vault:** Engaged users click the card to visit vaults.fyi, where they can explore detailed analytics, supply or withdraw funds in one click, track positions, and find additional opportunities.

    ### Search Criteria for Yield Opportunities

    To ensure only higher-signal opportunities are shown, the following criteria apply:

    - Total Value Locked (TVL) must be greater than $100 million.
    - For stablecoin vaults, only results where the underlying asset is USDC, USDT, or DAI are returned.

    The vault with the highest 7-day APY found for ETH and for stablecoins is surfaced. Simple criteria ensure that users see yield options with substantial liquidity and protocol maturity.

    ### API Capabilities Used

    - **Market data:** Real-time, structured data on yield opportunities.
    - **Portfolio & personalized insights:** Wallet balance data and relevant yield opportunity surfacing based on user preferences.
    - **Transactional support:** Frictionless DeFi interactions enabling users to supply, withdraw, or manage funds directly within integrated apps.

    Together, these API capabilities create a seamless, informed experience that empowers users to put their tokens to work.
